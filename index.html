八智能体角色定义与职责边界

1
线索评级Agent
Lead Scoring Agent
基于客户行为数据、企业画像、历史成交记录，对新线索进行多维度评分与分级（S/A/B/C/D），输出优先级排序。
INPUT 线索表单、官网行为、邮件打开率、企业工商信息
OUTPUT 线索评分(0-100)、等级标签、分配建议、跟进urgency
TOOL 企业征信API、行业数据库、历史成交模型
2
客户画像Agent
Customer Profile Agent
整合CRM、邮件、会议记录、合同等多源数据，动态生成360度客户画像，识别决策链关键人、预算周期、竞品使用情况。
INPUT CRM记录、邮件往来、会议纪要、合同历史、社交媒体
OUTPUT 客户画像JSON、决策链图谱、风险预警、关系热力图
TOOL NLP实体抽取、关系图谱构建、时间序列分析
3
知识检索Agent
Knowledge Retrieval Agent
通过RAG架构检索企业知识库（产品文档、竞品分析、成功案例、FAQ），为销售提供实时话术支持与方案推荐。
INPUT 销售当前场景描述、客户问题、产品需求关键词
OUTPUT Top-K相关文档、推荐话术、竞品对比表、成功案例
TOOL 向量数据库(Chroma/Milvus)、Embedding模型、重排序模型
4
任务规划Agent
Task Planning Agent
根据商机阶段、客户画像、历史最佳实践，自动生成跟进任务序列（拜访、方案、报价、谈判），并分配执行人与截止时间。
INPUT 商机阶段、客户画像、销售日历、团队负载
OUTPUT 任务DAG图、执行人分配、截止时间、依赖关系
TOOL 日历API、任务调度引擎、负载均衡算法
5
工具执行Agent
Tool Execution Agent
通过MCP协议调用外部工具（发邮件、建日程、生成PPT、查物流、调报价系统），将规划任务落地为具体操作。
INPUT 任务指令、工具参数、上下文状态
OUTPUT 工具执行结果、异常报告、状态变更
TOOL MCP Server、邮件API、日历API、文档生成API
6
质检Agent
Quality Assurance Agent
对销售跟进记录、客户沟通内容、方案文档进行质量检查，识别信息缺失、逻辑矛盾、合规风险，生成质检报告。
INPUT 跟进记录、沟通录音转写、方案文档、合规规则库
OUTPUT 质检评分、缺失项清单、风险提示、改进建议
TOOL 规则引擎、NLP一致性检测、合规知识库
7
审批守卫Agent
Approval Guard Agent
对关键节点（大额折扣、合同变更、客户转交）进行自动审批校验，匹配审批矩阵，触发多级审批流，记录审计日志。
INPUT 审批申请、金额阈值、客户等级、历史折扣率
OUTPUT 审批结果、驳回原因、升级路径、审计日志
TOOL 审批矩阵引擎、BPMN工作流、数字签名
8
CRM回写Agent
CRM Write-back Agent
将Agent协同产生的所有结构化数据（评分、画像、任务、质检结果）自动回写CRM系统，确保数据一致性与可追溯性。
INPUT 各Agent输出结果、CRM数据模型、字段映射规则
OUTPUT 回写确认、变更日志、数据一致性校验报告
TOOL CRM API(Salesforce/纷享销客/销售易)、ETL管道、CDC

商机流转协同链路（端到端）

1
线索录入触发
新线索从官网表单/展会/cold call进入系统 -> 线索评级Agent 启动评分，输出 S/A/B/C/D 等级
2
客户画像构建
客户画像Agent 拉取CRM历史、邮件、会议记录 -> 生成360度画像 + 决策链图谱 -> 识别关键决策人
3
知识检索与方案生成
知识检索Agent 通过RAG召回相关产品文档、成功案例、竞品分析 -> 生成定制化方案话术
4
任务规划与分配
任务规划Agent 根据画像+知识输出 -> 生成跟进任务DAG（拜访->方案->报价->谈判）-> 自动分配执行人
5
工具执行落地
工具执行Agent 通过MCP调用邮件发送、日程创建、PPT生成、报价查询 -> 自动执行规划任务
6
质检与合规检查
质检Agent 扫描跟进记录、沟通内容、方案文档 -> 识别信息缺失/逻辑矛盾/合规风险 -> 生成质检报告
7
审批守卫校验
审批守卫Agent 对大额折扣/合同变更/客户转交 -> 匹配审批矩阵 -> 触发多级审批流 -> 记录审计日志
8
CRM数据回写
CRM回写Agent 将评分/画像/任务/质检结果 -> 结构化回写CRM -> 确保数据一致性与全链路可追溯

状态机流转模型

# 商机状态机定义 (State Machine)
class OpportunityStateMachine:
    STATES = {
        "NEW":        "新线索",
        "SCORED":     "已评级",
        "PROFILING":  "画像构建中",
        "PLANNED":    "任务已规划",
        "EXECUTING":  "执行中",
        "QA_PASSED":  "质检通过",
        "APPROVED":   "审批通过",
        "WON":        "赢单",
        "LOST":       "输单",
        "HOLD":       "人工接管"
    }

    TRANSITIONS = {
        "NEW"        -> ["SCORED"],          # 线索评级Agent
        "SCORED"     -> ["PROFILING"],       # 客户画像Agent
        "PROFILING"  -> ["PLANNED"],         # 任务规划Agent
        "PLANNED"    -> ["EXECUTING"],       # 工具执行Agent
        "EXECUTING"  -> ["QA_PASSED", "HOLD"], # 质检Agent / 人工接管
        "QA_PASSED"  -> ["APPROVED", "HOLD"],  # 审批守卫Agent / 人工接管
        "APPROVED"   -> ["WON", "LOST"],      # 最终状态
        "HOLD"       -> ["EXECUTING", "LOST"] # 人工干预后恢复
    }

成本预算控制模型

# 成本预算控制伪代码
class CostController:
    BUDGET_PER_OPPORTUNITY = 2.0   # 美元/商机
    BUDGET_PER_SALES = 50.0       # 美元/销售/月

    MODEL_TIERS = [
        { "model": "gpt-4o",        "cost_per_1k": 0.005, "quality": 1.0 },
        { "model": "claude-3.5-sonnet", "cost_per_1k": 0.003, "quality": 0.95 },
        { "model": "qwen2.5-72b",   "cost_per_1k": 0.001, "quality": 0.85 },
        { "model": "local-llm",     "cost_per_1k": 0.0001, "quality": 0.7 }
    ]

    def select_model(self, remaining_budget, required_quality):
        for tier in self.MODEL_TIERS:
            if tier["quality"] >= required_quality and \
               self.estimate_cost(tier) <= remaining_budget:
                return tier["model"]
        return "local-llm"  # 兜底本地模型

    def check_budget(self, sales_id, opportunity_id):
        used = self.get_monthly_usage(sales_id)
        if used >= self.BUDGET_PER_SALES * 0.8:
            self.send_alert(sales_id, "预算使用达80%")
        if used >= self.BUDGET_PER_SALES:
            return "FORCE_DEGRADE"
        return "OK"

Agent编排核心代码

# LangGraph 多Agent编排示例
from langgraph.graph import StateGraph, END
from typing import TypedDict

class OpportunityState(TypedDict):
    opportunity_id: str
    current_stage: str
    lead_data: dict
    customer_profile: dict
    knowledge_results: list
    task_plan: list
    tool_results: list
    qa_report: dict
    approval_result: dict
    crm_writeback: dict
    confidence: float
    requires_human: bool
    cost_accumulated: float

# 定义Agent节点
def lead_scoring_agent(state: OpportunityState):
    score, grade = scoring_engine.evaluate(state["lead_data"])
    cost = token_tracker.track("lead_scoring")
    return {
        "current_stage": "SCORED",
        "confidence": score / 100,
        "cost_accumulated": state["cost_accumulated"] + cost
    }

def customer_profiling_agent(state: OpportunityState):
    profile = profile_builder.build(state["opportunity_id"])
    cost = token_tracker.track("profiling")
    return {
        "customer_profile": profile,
        "current_stage": "PROFILING",
        "cost_accumulated": state["cost_accumulated"] + cost
    }

def knowledge_retrieval_agent(state: OpportunityState):
    results = rag_engine.retrieve(
        query=state["customer_profile"]["current_needs"],
        top_k=5,
        namespace=state["opportunity_id"]
    )
    cost = token_tracker.track("retrieval")
    return {
        "knowledge_results": results,
        "cost_accumulated": state["cost_accumulated"] + cost
    }

def human_takeover_guard(state: OpportunityState):
    if state["confidence"] < 0.7 or state["cost_accumulated"] > 2.0:
        return {"requires_human": True, "current_stage": "HOLD"}
    return {"requires_human": False}

# 构建状态机图
workflow = StateGraph(OpportunityState)
workflow.add_node("lead_scoring", lead_scoring_agent)
workflow.add_node("profiling", customer_profiling_agent)

# MCP Server 注册与调用
class MCPToolRegistry:
    def __init__(self):
        self.tools = {}

    def register(self, name, schema, handler):
        self.tools[name] = {"schema": schema, "handler": handler}

    def execute(self, name, params):
        if name not in self.tools:
            raise ToolNotFoundError(name)
        return self.tools[name]["handler"](**params)

# Agent调用工具
def tool_execution_agent(state):
    plan = state["task_plan"]
    results = []
    for task in plan:
        tool_name = task["tool"]
        params = task["params"]
        try:
            result = retry_with_backoff(
                lambda: registry.execute(tool_name, params),
                max_retries=3
            )
            results.append({"task": task, "status": "success", "result": result})
        except Exception as e:
            results.append({"task": task, "status": "failed", "error": str(e)})
            if task["critical"]:
                return {"requires_human": True, "error": str(e)}
    return {"tool_results": results}

RAG召回链路设计

# RAG Pipeline: 混合检索 + 重排序
class RAGPipeline:
    def __init__(self):
        self.vector_store = MilvusClient(uri="milvus://localhost:19530")
        self.embedder = SentenceTransformer("BAAI/bge-large-zh-v1.5")
        self.reranker = CrossEncoder("BAAI/bge-reranker-large")
        self.bm25_index = BM25Index(corpus=knowledge_corpus)

    def retrieve(self, query: str, namespace: str, top_k: int = 5):
        # Step 1: 向量检索
        query_vec = self.embedder.encode(query, normalize_embeddings=True)
        vec_results = self.vector_store.search(
            collection_name=f"knowledge_{namespace}",
            data=[query_vec],
            limit=20,
            output_fields=["content", "source", "metadata"]
        )

        # Step 2: BM25关键词检索
        bm25_results = self.bm25_index.search(query, top_k=20)

        # Step 3: 结果融合去重 (RRF)
        fused = self.reciprocal_rank_fusion(vec_results, bm25_results)

        # Step 4: Cross-Encoder重排序
        pairs = [(query, doc["content"]) for doc in fused]
        scores = self.reranker.predict(pairs)
        for doc, score in zip(fused, scores):
            doc["rerank_score"] = score

        # Step 5: 取Top-K并上下文压缩
        top_results = sorted(fused, key=lambda x: x["rerank_score"], reverse=True)[:top_k]
        compressed = self.context_compressor.compress(top_results, max_tokens=4000)

        return compressed
